# Design Rules (Standard)

## Product Feel
- Keep the interface clear, trustworthy, and practical.
- Prioritize fast scanning, obvious actions, and low-friction flows.
- Avoid decorative UI that hides the actual service state or task.
- Avoid generic AI aesthetics (purple gradients on white, stock cards, template-like layouts) unless the brand explicitly calls for it.

## Responsive Rules
- Design mobile and desktop together.
- No horizontal overflow on mobile.
- Navigation, forms, and primary CTAs must remain usable on narrow screens.
- Use stable dimensions for repeated UI elements to prevent layout shifts.

## Components
- Use consistent spacing, typography, and button hierarchy.
- Include loading, empty, error, and disabled states.
- Keep copy direct and user-centered.

## Accessibility
- Prefer semantic HTML.
- Controls are keyboard reachable; focus and hover states are visible.
- Color contrast is reasonable.
- Icon-only controls have accessible labels or tooltips.
